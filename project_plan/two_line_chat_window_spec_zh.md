# 两条线交互 Spec

更新时间：2026-07-03

## 目标

把桌宠 Copilot 拆成两条清楚的产品线：

```text
自动观察线：
  小机器人持续观察窗口，展示摘要和候选问题。

用户对话线：
  用户提问后打开独立悬浮对话窗，展示历史对话、当前回答和输入框。
```

## 设计原则

- 小机器人保持轻量，不承载长对话。
- 独立对话窗是同一个桌面体验的一部分，不做网页式突兀窗口。
- 两条线共享本地 RuntimeStore 中的 latest analysis、conversation 和 session memory。
- KV cache 只做性能优化，不承担可靠记忆。
- 用户提问时暂停自动观察，避免回答过程中的上下文漂移。

## 自动观察线

小机器人窗口负责：

```text
状态表情
当前窗口摘要
关键点
候选问题
自定义提问入口
打开或聚焦对话窗
```

## 用户对话线

独立 Win32 悬浮对话窗与小机器人运行在同一 Python 进程中。

第一版能力：

```text
展示最近历史对话
展示当前流式回答
支持输入新问题
支持继续观察
可关闭但不退出主机器人
视觉风格与机器人一致
```

## 后端接口

```text
POST /api/assistant/questions
GET  /api/assistant/conversation
GET  /api/assistant/conversations
POST /api/assistant/resume
```

## 当前不做

- 不引入额外本地基础设施。
- 不做复杂多窗口布局管理器。
- 不做自动电脑操作。
- 不把 KV cache 当成 memory。
- 不在第一版切换模型问答链路。

## 验收

- 点击候选问题会打开独立对话窗，并开始回答。
- 自定义输入会打开独立对话窗，并开始回答。
- 对话窗能展示当前回答和最近历史。
- 对话窗关闭不影响小机器人。
- 点击“观察”后清空当前会话并恢复 watcher。
- 后端测试通过。
- 桌面窗口脚本通过 Python 编译检查。
